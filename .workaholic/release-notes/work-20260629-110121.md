# Wire the qfs binary so the docs run true (v0.0.9 → v0.0.12)

## Summary

This release turns qfs from a tool whose docs described features the binary didn't run into a wired, working product where the documentation runs true — reads across every service, an in-language connection model, cookbook-as-Agent-Skills, and a composable cross-service read pipeline.

## Key Changes

- **Wire-binary epic (9 tickets):** `/local`, `/sql`, `/git`, `/github`, `/slack`, `/mail`, `/drive`, `/ga` reads all return rows end to end; codec `decode`/`encode` execute; fresh users get an honest "connect your account" error instead of `unknown_source`.
- **In-language connections:** `CREATE CONNECTION … DRIVER … SECRET 'env:/vault:…'` replaces the env-var alias convention; `CONNECT`/`DISCONNECT` defined paths replace fixed driver mounts (bindings persist in the project DB).
- **Docs run true:** every guide/cookbook example verified against the binary; each cookbook article generates a Claude Code Agent Skill, gated by a parse-check ratchet.
- **FTP-parity + composable reads:** array/struct/bytes literal constructors, Gmail attachment byte-read, mkdir parity, and the composable `array_agg(struct)` read pipeline with a real per-row `EXTEND`.

## Changes

### Added
- In-language `CREATE CONNECTION` + `CONNECT`/`DISCONNECT` defined paths; live Postgres/MySQL/MinIO backends; `/cf` and `/rest` reachable; Agent Skills generation (`gen-skills`); composable `array_agg(struct)` read pipeline.

### Changed
- Wired real reads for all local and cloud sources; pushdown deepened (WHERE/ORDER BY/LIMIT/projection); documentation rewritten to shipped reality; `/ga` → `/google-analytics` (deprecated `/ga` alias).
