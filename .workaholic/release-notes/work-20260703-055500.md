# Request the OIDC identity scopes in the Google consent etc

## Summary

v0.0.16 fixes the final step of the paste-back Google consent: on v0.0.15 the flow failed with `userinfo status 401` because the consent requested only API scopes. The consent now also requests `openid email`, so the account's profile-email lookup succeeds.

## Key Changes

- The Google consent auth URL requests the OIDC identity pair (`openid email`) alongside the four least-privilege API scopes (gmail.modify, gmail.compose, drive, analytics.readonly)
- The scope union is test-pinned at exactly those six scopes; the broad `profile` and full-mailbox scopes are forbidden
- Re-running `qfs account add google` accumulates the new scopes onto an existing grant (`include_granted_scopes`), completing an onboarding interrupted on v0.0.15

## Changes

### Changed
- Consent scope union now includes `openid` and `email` so the userinfo profile-email lookup that keys the account no longer returns 401
