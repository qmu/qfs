# v0.0.20: Three blueprint designs made real (result envelope, path-aware PBAC, commit-boundary materialization), etc

## Summary

v0.0.20 hardens the credential vault (owner-only `0600` store files + a time-boxed session unlock),
then an overnight `/drive` lands three approved blueprint designs — the §14 schema-carrying result
envelope with `?limit`/`?offset` endpoint paging, §8 path-aware capability grants, and §7
commit-boundary write materialization — and fixes two "describe never lies" correctness bugs found by
driving the built binary.

## Key Changes

- Three blueprint designs made real: the §14 result envelope (one serializer for `--json`/HTTP/MCP,
  base64 bytes) with endpoint paging; §8 path-aware `CapabilitySet` grants (data-only vs read-only
  DDL/DML matrix); and §7 commit-boundary materialization that makes `… |> upsert into <dst>` blob
  copy actually commit.
- Real correctness bugs found by driving the binary: `REMOVE TABLE` was rejected in one-shot mode, and
  Gmail advertised collection writes it refused at commit — both fixed with the honest fail-closed
  contract restored.
- Vault hardening: store DBs are created owner-only `0600` and re-verified on every open; one
  passphrase entry now skips re-prompts for 8h via a time-boxed session unlock.

## Changes

### Added

- Boolean literals (`where flag == true`) + the §14 schema-carrying result envelope `{schema, rows, meta}`
  with base64 bytes, one serializer for `--json` and the HTTP endpoint.
- `?limit`/`?offset` endpoint paging (shared `meta` vocabulary) + materialized-view `last_run` as
  readable data on `/server/views`.
- Console bundle delivery machinery (`crate::console`): pinned coordinate, sha256 verify, atomic cache,
  `QFS_UI_URL` override, same-origin CSP — behind an injectable fetcher seam (pin currently unset).
- Time-boxed session-unlock cache (one passphrase entry, 8h TTL) and owner-only `0600` store files with
  fail-closed loose-permission rejection.

### Changed

- Re-versioned the qfs plugin `0.1.0 → 0.2.0` so stale installed caches pick up the regenerated skills.
- Fixed `REMOVE TABLE` one-shot addressing and verified the SQLite DBMS surface end-to-end.
- Runtime capability grants are now path-aware (`PathScope` segment-glob) — a data-only policy admits
  table DML while denying catalog DDL (blueprint §8).
- Swept 1352 stale RFD citations in crate docs to blueprint anchors; deleted RFD-0001.
- Reconciled Gmail collection writes so `describe` never lies at commit (exact-id writes committable,
  set-wide predicate writes refused closed).
- Commit-boundary materialization for pipeline-sourced writes (blueprint §7) — blob copy commits.
