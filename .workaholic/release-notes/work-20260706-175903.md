# qfs FAQ skill, CLI-surface anti-drift, and git → path_binding convergence

## Summary

This release adds a comprehensive operator **FAQ Agent Skill** (`qfs-faq`) that answers how-do-I and troubleshooting questions — connecting an account, a second or different-organization Google account, the `org_internal` "access blocked" wall, the describe→preview→commit loop, common errors, and exit codes — directly from the skill, without reading qfs source. A new CI check keeps the FAQ's shell-command answers true as the CLI changes, and `git` connections now converge onto the canonical `qfs connect` (`path_binding`) registry like `/sql` already did.

## Key Changes

- **`qfs-faq` operator FAQ skill**: authored as `docs/cookbook/faq.md`, rendered to `qfs-faq/SKILL.md` by the existing `gen-skills` pipeline (no generator change). Covers connection & account setup (including a second / cross-org Google account and the `403 org_internal` consent-screen fix), the safety loop, common errors, the stable exit-code table, and a per-service routing index.
- **FAQ CLI-surface anti-drift check**: `crates/cmd/tests/faq_cli_surface.rs` asserts every `qfs <command> [--flags]` the FAQ cites exists in the real clap surface (via a new `qfs_cmd::clap_command()` reflection entrypoint), so a renamed or removed flag fails CI until the FAQ is corrected — closing the gap the `qfs`-recipe ratchet does not cover.
- **git → `path_binding` convergence**: a persisted `qfs connect /git/<repo>` binding now wires the runtime git driver and the `qfs describe` mount, so run / commit / describe read one source instead of only the env-var / `connections.qfs` seam.

## Changes

### Added
- `qfs-faq` FAQ / operator-reference Agent Skill (`docs/cookbook/faq.md` → generated `SKILL.md`), registered in the marketplace and linked from the docs cookbook; plugin versions bumped `0.2.0 → 0.3.0`.
- FAQ CLI-surface anti-drift test walking the clap `Command` tree (`qfs_cmd::clap_command()`), with a floor on extracted commands and negative-proof guards.

### Changed
- `git` connections converge onto the canonical `qfs connect` (`path_binding`) source for run, commit, and describe.
