# Dependency-posture decision log, dead-dep removal, and the serde_yaml exit

## Summary

This release runs the **Conservative Vendor Dependence** audit to completion and lands its durable
outcome: a per-dependency **decision log** in the living design doc (`docs/blueprint.md` §11), with a
Reason / Assessment / Monitoring / Exit record for the load-bearing dependency set grouped by the
criterion that admits each one. It also takes the two hygiene actions the audit actually justified —
removing a dead `anyhow` declaration and exiting the archived/unmaintained `serde_yaml` for its
maintained community fork. No query-language surface changes; no plugin re-version.

## Key Changes

- **Dependency decision log (`docs/blueprint.md` §11)**: logs the taken dependencies by admitting
  criterion — error-is-fatal/far-from-core (crypto, `time`, `keyring`, `rustix`), interop/protocol
  conformance (`reqwest`, `url`, the DB drivers, `csv`), cost/time efficiency (`serde`, `toml`,
  `clap`, `tracing`, `tokio`, `winnow`), and parked ergonomics (`thiserror`, `async-trait`, `base64`,
  `bytes`, `rpassword`) — each with Reason / Assessment / Monitoring / Exit, satisfying the policy's
  "leave it in the repository" requirement.
- **Dead `anyhow` removed**: deleted from `[workspace.dependencies]`; it was declared but no member
  ever opted in.
- **`serde_yaml` → `serde_yaml_ng`**: `serde_yaml` was archived by its author at `0.9.34+deprecated`
  (2024); swapped to the maintained community fork, an API-compatible drop-in confined to
  `qfs-codec`. The vendor type stays internal to the codec (funnels through the shared
  `serde_json::Value` bridge), so the swap is invisible to the query language.

## Changes

### Changed
- YAML codec now uses the maintained `serde_yaml_ng` fork in place of the archived `serde_yaml`
  (`crates/codec`), behind the unchanged `qfs-codec` boundary.

### Removed
- Dead `anyhow` workspace dependency declaration.

### Documentation
- `docs/blueprint.md` §11 gains the per-dependency decision log.
