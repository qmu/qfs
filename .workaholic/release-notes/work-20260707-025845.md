# qfs configuration history and recovery surfaces

## Summary

qfs now treats configuration as inspectable, replay-aware operational state. This release adds a secret-free DDL/config event log, deterministic dump/restore, labeled Google OAuth apps, and updated current-design documentation.

## Key Changes

- Records secret-redacted DDL/config events for System DB-backed configuration writes.
- Adds `qfs dump` and preview-first `qfs restore` for deterministic JSONL backup and recovery.
- Supports labeled Google OAuth apps so cross-organization accounts use the right client credentials.
- Fixes `/gdrive` mount routing for describe and commit.
- Reorganizes qfs docs and generated skills around the current design snapshot.

## Changes

### Added
- DDL/config event log, `qfs dump`, and `qfs restore`.

### Changed
- Google app/account setup now requires explicit app labels.
- qfs documentation and plugin metadata now match the `0.0.27` design.
