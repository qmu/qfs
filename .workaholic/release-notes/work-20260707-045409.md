# qfs live runtime and provider commits

## Summary

qfs now refreshes materialized views explicitly, decodes richer Postgres values, and wires live Cloudflare and Drive commit paths while keeping unavailable providers fail-closed.

## Key Changes

- Materialized views can be refreshed, cached, and stamped with `last_run` only after success.
- Postgres NUMERIC, timestamp, UUID, and JSON values decode into qfs text values.
- Cloudflare D1/KV/Queues and Drive uploads now route through live read/commit boundaries.

## Changes

### Added
- `qfs view refresh` for materialized-view row caching and freshness stamps.
- Environment-backed Cloudflare D1, KV, and Queue read/commit wiring.

### Changed
- Postgres value decoding now covers richer scalar types.
- Interactive Drive copy commits now use the real applier and source-byte materialization.
