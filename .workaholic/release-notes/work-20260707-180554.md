# Declarative provisioning: qfs plan / qfs apply

## Summary

qfs now provisions itself like Terraform: one canonical `.qfs` document is the source of truth,
`qfs plan` shows the honest add/change/destroy diff against the live configuration, and
`qfs apply` converges both configuration stores — including the running daemon's live bindings —
idempotently and behind the existing safety gates. The blueprint also gains the `transform`
design (Decision W), and its grammar seam ships parse-only.

## Key Changes

- `qfs plan <document.qfs>` diffs desired vs. live config (exit 0 no-changes / 2 changes-pending);
  `qfs apply` reconciles: absent rows are REMOVEd behind `--commit-irreversible`, a moved base is
  refused without `--allow-stale-base`.
- The `/server` half applies through the daemon's existing public statement bridge — commits run
  inside the daemon, bindings reconcile live, and the boot config is re-emitted atomically so an
  applied reconcile survives restart.
- Secretish settings are excluded from the document universe entirely, and `qfs restore` no longer
  writes literal `<redacted>` over live secretish settings (shipped-flaw fix).
- Blueprint §15 (Decision W) reverses "qfs never calls an LLM" into a bounded, gated `transform`
  seam; the contextual-identifier grammar lands with the keyword set still frozen at 39.

## Changes

### Added
- `qfs plan` / `qfs apply` CLI verbs and the `qfs-provision` crate (canonical emitter, loader,
  set-difference diff, batch plan builder, store-routing applier).
- A `/server` read facet in the serve composition and live-applier commit routing with runtime
  audit, reconcile, and boot-config re-emission.
- Blueprint §15 `transform` (Decision W) and §16 provisioning (Decision X, including the face and
  process-boundary rulings); the `transform` pipe-stage grammar seam (execution deferred).

### Changed
- `qfs restore` skips secretish/`<redacted>` setting values on replay (counted), instead of
  clobbering live secrets.
- A non-loopback bind without bearer material refuses the commit bridge fail-closed.
- Plugin version `0.4.2 → 0.5.0`; qfs `0.0.31 → 0.0.32` (concurrent branches shipped 0.0.30/0.0.31 mid-flight).
