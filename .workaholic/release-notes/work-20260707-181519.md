# Cloudflare joins the qfs CONNECT model + a declared /cloudflare driver

## Summary

Cloudflare now uses qfs's uniform connection model: the compiled `/cf` driver is vault-backed with live resource discovery, its account id is auto-discovered on connect, and a new declared `/cloudflare` driver covers the broad REST surface with no compiled Rust.

## Key Changes

- `/cf` migrated off `CF_*` environment variables to a vault account plus `path_binding`, with D1/KV/Queues discovered live at mount time.
- `qfs connect /cf` auto-discovers the Cloudflare account id from the token, so `--at` is optional (fails closed on a multi-account token).
- New declared `/cloudflare` driver (`cloudflare.qfs`): zones, DNS records, and account-scoped KV/Queues/D1 listings, host-confined and credential-free, live-verified against a real account.

## Changes

### Added
- Declared query-language `/cloudflare` driver plus a `qfs-cloudflare` cookbook skill.

### Changed
- `/cf` moved to the `qfs account` / `qfs connect` model with live resource discovery.
- `qfs connect /cf` makes `--at` optional via account-id auto-discovery.
