# Implement the Transform model-call seam end-to-end etc

## Summary

qfs v0.0.42 ships the Transform feature — model-calling pipeline stages behind a single enforced seam — alongside the type-system foundation (refined types, two-layer model, arithmetic operators), Cloudflare Artifacts repositories, and a fix for the SQLite database-locked test flake.

## Key Changes

- CREATE TRANSFORM end-to-end: definition DDL and storage, plan spine, execution through the ModelProvider seam with three cardinality modes, and a fully documented, parse-checked taught surface
- Type-system foundation: blueprint type chapter, CREATE TYPE WHERE refinements, CREATE TABLE OF write-boundary contracts, canonical type-annotation vocabulary
- Language tightening: arithmetic operators with monomorphic semantics, unified stdlib name resolution, the reference convention (paths = data, names = definitions), and a lock asserting transform is the only model-call seam
- Cloudflare Artifacts Git repositories as a /cf/artifacts resource with read-after-write and vault-sealed tokens
- SQLite "database is locked" flake fixed by arming busy_timeout before the WAL pragma

## Changes
