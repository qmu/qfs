# Capability tryout: switch routing, live providers, cross-service files, declared-driver proofs

## Summary

qfs v0.0.56 completes the capability-tryout mission's implementation arc across thirteen tickets.
The headline is model-chosen routing in the pipe language ‚Äî `|> switch <col> { 'a' => <arm>, else =>
<arm> }` routes rows to pre-declared, operator-consented arms by a transform's output, so the model
chooses among consented effects and can never invent one ‚Äî proven in an owner-attended live round
that committed real Drive and Gmail writes routed by a real Anthropic model. Around it: three live
model providers with real PDF document input, cross-service file transfer in both directions plus
reply-with-attachment, real Slack bytes upload, two declared-driver rewrite proofs (API-key Chatwork,
account-auth GitHub), restored in-server CREATE JOB firing, a machine-enforced command-execution
lock, and a re-measured dependency posture. The live round also surfaced and fixed a latent panic
that had silently blocked every live `|> transform` commit.

## Key Changes

- `|> switch` routing: a transform emits a discriminant column, rows route to declared sub-pipeline arms (writes or CALLs) in declaration order; the model runs once, untaken arms are previewed-but-not-fired, one apply commits ‚Äî live-proven with exact per-arm counts
- Three live `ModelProvider` impls (Anthropic, OpenAI, Google) behind the one-seam transform lock, now with real per-provider PDF/image document input for Extraction mode and pre-network size caps
- Cross-service file story complete: Gmail-attachment ‚Üí Drive-folder in one statement; Gmail reply threaded with a Drive-sourced attachment via the new `/mail/<label>/<msg>/replies` append-log; real Slack bytes upload over the external-upload flow
- Declared-driver rewrite proofs: `chatwork.qfs` (API-key, zero compiled code) and the new generic `AUTH ACCOUNT '<provider>'` clause with a `github_account.qfs` proof mounting `/ghdecl` beside the compiled driver
- In-server `CREATE JOB` firing restored (t65 reversed): pure injected-clock cadence decisions, missed-fire/overlap/UTC semantics ruled, firing rides the existing watchtower commit path
- Command-execution posture is now an enforced invariant: an exact allowlist test over every process-spawn site, shell-string execution forbidden, git argument injection proven neutralized
- Transform chains pinned true-by-test (per-stage typing, model-free preview, atomic mid-chain abort); dependency posture re-measured (30 shipped direct deps, removable-today ‚âà 0)
- Live-round defect fixes: driver-http ambient-runtime panic (was blocking every live transform commit), switch prune now bridges dependency edges (declaration order + fail-stop restored), System-DB v16 in-place edit self-heals
