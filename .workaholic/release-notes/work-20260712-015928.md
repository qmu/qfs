# Honest counts on the write side: multi-row Drive uploads, loud declared-type refusal

## Summary

qfs v0.0.58 fixes the two driver defects the owner-attended switch live round surfaced. A
multi-row `INSERT INTO /drive/<folder>` now creates one file per row with `affected` equal to
files actually written (it previously wrote only the first row while reporting every row as
affected), and a declared view whose `OF` type resolves no columns — the stale pre-§5.4 type-row
case that made live Chatwork reads return value-less rows — now refuses loudly before any wire
fetch, with re-install genuinely healing the mount.

## Key Changes

- Multi-row Drive INSERT/UPSERT decodes one upload per row; the whole batch validates before the first write; a mid-batch API failure surfaces as a structured `partial_apply` error naming exact progress, never full success
- Multi-row UPDATE/REMOVE/CALL against Drive fail closed instead of silently dropping extra rows
- The batch's shared destination folder resolves once (memoized walk), not once per row
- A declared `OF` contract resolving zero columns refuses pre-network with a structured `invalid_path` naming the fix (re-install the driver declaration)
- Declared-type lookup prefers the newest same-name declaration, so re-installing a declaration heals a stale mount (matches the describe path's ordering)
- The cross-service cookbook now states the missing-folder failure honestly: path-shape errors at preview; folder existence is a `not_found` at commit (preview performs no network reads)

## Changes

### Changed

- driver-gdrive: per-row effect decode (`from_node_rows_with`), batch-validating applier with `DriveError::PartialApply`, `MemoResolver` destination memoization; 4 new hermetic locks including the live-round 2-row shape verbatim
- qfs-exec declared views: `eval_view_body` refuses `of_columns == []` before the fetch; 2 new seam tests including a real-shaped Chatwork `/rooms` bare-array fixture with values asserted present
- qfs binary: `types_from_conn` orders `id DESC` (newest declaration wins); 3 new tests pinning the stale array-body parse, the `Some([])` encoding, and newest-wins healing
- Cookbook/skill: cross-service upload recipe's preview/commit failure split corrected (plugin 0.11.1)
