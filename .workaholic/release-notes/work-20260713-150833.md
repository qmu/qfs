# v0.0.60: fix the 8 live-round defects (sweeper, Drive/Slack/GitHub reads, /local content, reasoning models)

## Summary

qfs v0.0.60 fixes the eight defects the owner-attended live rounds surfaced ‚Äî silent wrong-answer,
wrong-node-write, and never-worked paths the hermetic suite could not reach. Each fix lands with a
hermetic lock (several verified by a negative test). Headline: the v0.0.59 daemon sweeper now
actually fires a `CREATE JOB` (it never had), a Drive folder `UPDATE` no longer renames the folder,
and a Slack/GitHub `WHERE` no longer returns the whole collection.

## Key Changes

- **Sweeper fires**: `fire_one` rehydrates the stored canonical `PlanSpec` (`from_canonical`, no
  re-parse) instead of re-lexing AST-JSON as statement text ‚Äî a live-committer test writes a real
  tick file.
- **Drive folder `UPDATE`**: a name-path folder rename refuses loudly (the WHERE-drop that renamed
  the container is gone); id/`file_id` addressing still renames the folder.
- **Slack/GitHub `WHERE`**: the read facet enforces the pushed predicate at the seam, so a filter
  can never silently return the unfiltered set.
- **`/local` `content`**: `describe()` advertises the nullable `content` column, so
  `|> select content |> transform` and the PDF-extraction recipe type-check at plan time.
- **Gmail read-terminal transform**: source reads are stripped before the consent-ledger apply, so
  a pure-read transform chain over Gmail commits.
- **Reasoning models**: OpenAI sends `max_completion_tokens`, effort ceilings leave post-thinking
  room, empty completions get an actionable error.
- **`/drive/id:<file-id>`**: the mounted id form parses, so a space-named file is reachable by id.
- **Drive folder `UPSERT`**: per-row named decode (replace-or-create), so the INSERT-collision
  advice works.
