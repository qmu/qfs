# Fix Slack file detach and Gmail attachment addressing etc

## Summary

v0.0.62 fixes two file-handling defects found in owner-attended live rounds: deleting a Slack file
now works, and a Gmail attachment can be transferred straight into a Drive folder in one statement.
Both are live-proven byte-exact.

## Key Changes

- **Slack file detach** now works: `remove /slack/<ws>/files/<id>` deletes the file (`files.delete`). The upload/list/download/delete loop is complete.
- **Gmail attachments** are addressed by a stable index — `/mail/<label>/<msg>/att0` (Gmail's internal attachment id is ephemeral, so qfs resolves the live id for you inside the read). This makes a one-statement Gmail→Drive transfer work byte-identical.

## Changes

### Changed
- Slack: the file node (`/slack/<ws>/files/<id>`) now supports `remove`, decoding to an irreversible `files.delete`.
- Gmail: attachment addressing moved from the (ephemeral) raw id to the stable `att<N>` index, resolved fresh inside the read.
