# Sample output in the Cloudflare cookbook is now a neutral example

## Summary

qfs v0.0.71 is a housekeeping release ahead of publishing the project. There is no functional change: the binary behaves exactly as v0.0.70 did. The Cloudflare cookbook's sample `/cloudflare/zones` output had been captured from a real account rather than written as an example, and the same lines had been carried into the shipped `qfs-cloudflare` Agent Skill. Both now show neutral example zones. Test fixtures and internal engineering records were swept on the same principle.

## Key Changes

- `docs/cookbook/cloudflare.md` and the generated `qfs-cloudflare` skill show example zones (`acme.com`, …) instead of a real account's zone list. The recipe, the shape of the output, and the row count are unchanged — only the values.
- Plugin version `0.11.7` → `0.11.8` so installed caches pick up the regenerated skill. No taught surface changed, so this is a patch bump.
- The Gmail driver's label-listing test keeps every property it exercises — a user label, a non-ASCII display name, `id != name` — under a synthetic label name.

## Changes

### Changed
- Cloudflare cookbook and `qfs-cloudflare` skill: sample zone output is now a neutral example.
- Gmail driver: the label-listing test fixture and its doc comments use a synthetic label.
- Engineering records under `.workaholic/`: provenance asides that named a dev box's sibling repositories, or values read from live accounts during verification, now state the claim without the aside. What was verified, and that it was verified live, is unchanged.
