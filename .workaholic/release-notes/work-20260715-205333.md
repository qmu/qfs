# Unify the qfs statement splitter onto the lexer etc

## Summary

qfs v0.0.72 unifies the `.qfs` statement splitter onto the real lexer, fixing a shipped parse bug where statements merged silently, and makes re-installing a declaration replace its registry row so append-era duplicates heal on read.

## Key Changes

- One lexer-backed statement splitter in qfs-core replaces both hand-rolled scanners; `--`, `#`, and `;` inside paths and quoted locators are now treated as content, and `transaction { … /path; … }` parses correctly
- Hard break: a bare (unquoted) path can no longer carry a literal `;` — quote the locator if you need one (plugin 0.11.9)
- Re-installing a declaration now replaces its `(kind, name, verb)` row inside the audited transaction; reads resolve newest-per-key, so registries with append-era duplicates heal without re-install
- Missions are reframed as standing product properties with per-ticket verification gates

## Changes

### Changed
- Unified the `.qfs` statement splitter onto the lexer; deleted both hand-rolled scanners and added `;` to the path-delimiter set (deliberate hard break)
- Re-install now replaces the same-key declaration row; reads resolve newest-per-key
- Reframed the active missions as standing product properties and demoted their gates per owner ruling
