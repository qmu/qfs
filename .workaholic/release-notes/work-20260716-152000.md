# Re-home the declarative tables into the System DB etc

## Summary

qfs v0.0.73 moves the declarative config registry (`path_binding`, `connection_consent`) out of the credential vault and into the System DB beside the ledger, so every configuration write is recorded atomically — row, audit event, and replayable DDL event in one transaction.

## Key Changes

- Every `CONNECT`/`DISCONNECT` and account declare/remove now lands in the hash-chained DDL event log, in the same transaction as the write — the config history is finally queryable
- Existing installs heal automatically: a one-shot boot copy moves legacy rows into the System DB, recording itself in the ledger as its own idempotency marker
- The Project DB is now the vault proper — one file holds secret material, the other holds everything declarative plus the ledger that observes it
- `qfs dump` gains the previously-missing accounts/consent section, and `qfs restore --commit` of bindings and consents now records local audit and DDL events

## Changes

### Changed
- Re-homed `path_binding` and `connection_consent` into the System DB (migration #17) with a marker-guarded one-shot boot copy; all readers repointed
- Converged every config writer on shared ledger-transactional helpers (statement path, CLI, and restore land the same shape)
- Completed the v0.0.72 carry handoff (report, ship, and release confirmation)
