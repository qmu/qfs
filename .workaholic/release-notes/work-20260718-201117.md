# Slack write fixes, tmux safe-teardown, and the claude session surface etc

## Summary

Ships the hermetic half of the Claude-sessions mission: the readable /hosts/local/claude session surface, Slack driver write-path fixes (channel-id resolution everywhere, user-token DMs), a TmuxSocket isolation primitive whose teardown can never kill the developer's server, the launcher-id banner-parsing fix, and the schema-agnostic teams-inbox steering medium. The steering live-drain and launch proofs stay iceboxed for attended work.

## Key Changes

- Sessions queryable: `/hosts/local/claude/sessions` reads the real ~/.claude store with liveness detection and truthful last_message
- Slack channel names and user IDs resolve identically across all verbs (delete, update, reactions, pins, upload); user-token DMs via conversations.open
- TmuxSocket isolation primitive (`-L` enforced at the type boundary) and a launcher that returns a real session id from the 2.1.217 banner

## Changes
