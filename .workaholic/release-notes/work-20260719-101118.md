# Thread the request principal to the query path etc

## Summary

Delivers "who am I" on the path a query actually takes: every request resolves to a named principal or the first-class not-signed-in answer, /sys/whoami reads it as data, session cookies resolve to a UserId on serve, and the policy gate evaluates the real actor. Fail-closed defaults are preserved and test-pinned.

## Key Changes

- Principal enum + RequestContext threaded to `ReadDriver::scan` (the M2 seam, every driver updated) — principal receipt is a compile-time fact
- `/sys/whoami` closed-set facet with credential-free `{signed_in, user}`; session-cookie → UserId resolution wired on serve, fail-closed
- Policy gate evaluates the resolved actor via `evaluate_with_context`; identity::Role pinned as NOT a grant

## Changes

### Added
- `Principal` / `RequestContext` in qfs-core; `/sys/whoami` facet; serve-side /sys read registration; session-cookie principal resolver; `qfs identity whoami --json`

### Changed
- `ReadDriver::scan` signature carries the request context (hard break); HTTP policy gate evaluates DecisionContext
