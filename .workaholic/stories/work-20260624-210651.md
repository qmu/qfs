# Branch Story: work-20260624-210651

## 1. Overview

Closes the post-install onboarding gap: after a successful `install.sh`, qfs now prints a short
**Next steps** welcome so a first-time user can immediately tell it's installed and knows what to do.
Implements the `post-install-onboarding-guidance` ticket. Base: `main`. Patch bump `0.0.2 → 0.0.3`.

## 2. Changes

- **`packages/qfs/install.sh`** — a `next_steps()` block printed after a verified install (and after
  `--version`), covering: try it (offline, no account — `qfs describe`, `qfs run` PREVIEW), connect a
  service (`qfs account add`, credential never printed), update (re-run the installer; pin with
  `QFS_VERSION`), and docs (GitHub README + getting-started, plus `qfs skill`).
- **`README.md`** — one-line pointer in the Install section noting the installer prints Next steps;
  the existing Quickstart is the same loop.
- **Version** — `0.0.2 → 0.0.3` (per the per-PR patch rule).

## 3. Outcome

- Verified end-to-end: ran the updated `install.sh` against the live release; the welcome renders

- `sh -n` clean; `qfs --version` → `0.0.3`; credential gate passes on the new block; anti-drift
  `gen-docs --check` in sync.

## 4. Concerns

None. Terminal output + a one-line README note + version string; no CLI behavior change. A future,
