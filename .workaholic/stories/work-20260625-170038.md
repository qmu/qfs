## 2. Changes

- **`docs/roadmap.md`** — the vision + phased plan, extended this cycle with decisions **R–V**:
  - **R** — a `/path` is a first-class `Resource` value; the source position drops `from`; one spelling
    for "a node" (also `join`/set-op operands, `policy` targets, `member_of`). Leading `/` is
    unambiguous by **position** (path at expression-start, division infix), so **infix arithmetic
    (`+ - * /`) is supported** — this replaced the earlier, self-contradicting "no infix arithmetic"
    justification.
  - **S** — keywords are lowercase. **T** — a static primitive type system (`bool`,
    `i32/i64/u32/u64`, `f32/f64`, `string`, `Resource`) checked at plan time.
  - **U** — credential vault: three-level key hierarchy, team sharing by `policy` (never copy the
    secret), rotation/revocation, optional per-recipient E2E wrap.
  - **V** — observability externalized (emit, don't store): three sinks `file` (default) / `stdout` /
    `OTel` (recommended); hash-chained `/sys/audit` sealed to an external WORM/transparency log.
  - Plus a DDL clause-continuation carve-out to the `|>` rule, the SQL **connection-label** fix
    (host/user/db live in the credential, not the path), and migration of the 🧭 proposed examples to
    the target grammar (`§1.1` stays in today's grammar by design).
- **`docs/query-cookbook.md`** (new) — milestone-tagged worked recipes (`grammar=core|extended`).
- **`packages/qfs/crates/test/tests/roadmap_cookbook.rs`** (new) — parse-coverage ratchet:
  `grammar=core` recipes must parse with today's parser; `BASELINE_CORE` may only grow. (Internal
  naming corrected from the stale "roadmap"/"Part 7" wording to query-cookbook.)
- **Tickets `t42–t81`** + **`ROADMAP-TICKETS.md`** — the M0→M+ backlog. This cycle added the
  grammar trio **t73** (`Resource`/drop-`from`), **t74** (lowercase keywords), **t75** (type system),
  the audit/telemetry tickets **t76** (M0 hash-chained audit), **t77** (M3 sinks), **t78** (M3
  sealing), and the vault tickets **t79** (rotation/revocation), **t80** (E2E wrap), **t81**
  (self-hosted team-shared connections); removed the phantom `DEF` keyword; promoted observability
  from an M+ candidate to confirmed.
- **`.vitepress/`** — query-cookbook nav + nested outline; cloudflared tunnel host allow-list.
- **Version** — `0.0.7 → 0.0.8`.

## 4. Concerns

- **Docs/tickets run ahead of the binary — by design.** The roadmap is direction, not documentation;
  the cookbook is explicitly flagged and the ratchet test keeps `grammar=core` honest. No capability
  is claimed as shipped that isn't.
- **Open product decisions are flagged in-ticket, not guessed** — keyword case policy (t74), DB/passphrase
  UX (t42/t43), payment provider (t67), admin-page shape (t53).
- **One deliberate reversal:** infix arithmetic is now allowed (was "function-form only"). The cost is
  context-sensitive `/` lexing (path-vs-divide), called out in t73/t75; the benefit is SQL-familiar
  math and consistency with the cookbook.
- No carry-over concerns from prior drives apply (this is a planning branch; the E0–E8 foundation it
  builds on is already shipped).
