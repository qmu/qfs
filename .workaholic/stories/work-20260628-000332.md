# Branch Story: work-20260628-000332

## 1. Overview

Implements **the entire qfs roadmap** in one night `/drive`: all **40 roadmap tickets `t42–t81`**
(milestones **M0 → M9 plus M+**), the **onboarding `QFS_PASSPHRASE`** docs fix, and the remaining
pre-roadmap **driver-execution wiring** (gmail/gdrive/ga + s3/r2 live commit, github/slack networked
reads). Four older pre-roadmap tickets were dispositioned as superseded/done (the prior credential
foundation they assumed was replaced by M0). The branch turns qfs from a describe + preview engine
with a file-vault credential store into a SQLite-backed, identity-aware, OAuth-AS, three-faced
(CLI · MCP · dashboard) control plane with a full team/cloud/language surface. Base: `main`. Patch
bump **`0.0.8 → 0.0.9`** (one bump for the batch, per the per-PR rule; the deliverable is the
published release). **1800 tests, full gate green.**

The work was driven ticket-by-ticket: a subagent implemented each, the main agent independently
verified the full green bar (build/test/clippy/fmt/gen-docs/dep_direction + wasm) and committed +
archived each via the `workaholic:commit` convention. **89 implementation/archive commits + 1
version bump = 90 commits.**

## 2. Changes (by milestone)

- **M0 — Persistence (t42, t43, t44, t76).** New `qfs-store` leaf: SQLite **System DB + Project DB**
  with a forward-only, checksum-verified, transactional embedded-migrations runner (System now at
  migration v12, Project at v7). **Envelope-encrypted** credential store replaces the file vault
  (DEK sealed under an argon2id passphrase KEK; `account`→`connection` rename across the tree).
  Hash-chained **audit emission** on commit (emit-don't-store; only the chain head is durable).
- **M1 — Identity (t45, t46).** New `qfs-identity`: `users` + `accounts` tables, local argon2id
  sign-up, `qfs identity signup/whoami`. New `qfs-session`: CSPRNG session tokens stored hashed,
  expiry + rotation + revocation.
- **M2 — Server-as-MCP + OAuth AS (t47–t50).** New `qfs-mcp`: JSON-RPC MCP server (`describe`/
  `preview`/`commit`/`connections` tools) over the existing listener. New `qfs-oauth`: RFC 9728 PRM
  + RFC 8414 AS metadata + JWKS, **ES256** signing keys (RustCrypto `p256`), dynamic client
  registration (RFC 7591) + auth-code + **PKCE (S256-only)**, and **bearer + refresh tokens**
  gating `/mcp` (401 + `WWW-Authenticate` → discovery). The end-to-end handshake is real.
- **M3 — Dashboard (t51, t52, t53, t77, t78).** Embedded SPA shell (read-only) → preview→commit
  **approval cards** (one shared `commit_plan` path with MCP) → new `qfs-driver-sys` exposing
  `/sys/*` (users/projects/audit/connections/policies/metrics/settings/billing) as paths →
  externalized telemetry sinks (file/stdout/OTel-seam) → audit-chain **WORM sealing** (signed ES256
  checkpoints).
- **M4 — Cloud tier (t54).** Cloud drivers require a signed-in + consented connection (consent
  ledger, Project mig v3); fail-closed otherwise.
- **M5 — Self-hosted teams (t55–t59, t79–t81).** Invites + membership; upstream **OIDC federation**
  (verify + account-link, hermetic); extended **ACL** (actor/role/scoped-path/`member_of`, default-
  deny, no new keyword); `/directories` driver wiring the live `member_of` resolver; **3 selectable
  AI safety modes** (live on every commit path); credential **rotation/revocation + DEK re-wrap**;
  **per-recipient E2E** DEK wrap (p256 ECDH); **team-shared connections** (gate-before-decrypt).
- **M6 — Language (t60–t62, t70–t75).** `let` + `;`-free multi-statement programs; lambdas +
  `map`/`filter`/`reduce` (functions are values, no `DEF`); reversible-only `transaction`; `=` binds
  / `==` compares; path scope realms; writes-as-pipeline-stages; dropped `from`; **lowercase keywords**
  (case-insensitive, lowercase canonical); **static primitive type checker** active in production
  `build_plan`. Cookbook core parse-coverage rose 112 → ~350 across the grammar pass.
- **M7 — Agent fabric (t63, t64).** `qfs-tunnel` outbound-tunnel protocol core (framing/auth/mux);
  `/claude` session driver (decision K honored — no LLM dep).
- **M8 — Scheduling (t65).** Retired the internal scheduler daemon (deleted `qfs-cron`); `qfs job
  run`/`qfs job cron` external invocation (OS cron / Cloudflare Cron Triggers).
- **M9 — Managed team (t66, t67).** qfs Cloud OAuth **brokering** + team connections; free/paid
  **billing tiers** with entitlement gating (fail-closed-to-free).
- **M+ (t68, t69).** First-class **`fs` driver** (real filesystem blob namespace, deny-all-by-default
  named-root allowlist, path-escape guard); expansions-umbrella candidate ledger (docs-only).
- **Older / driver wiring.** onboarding `QFS_PASSPHRASE` prerequisite documented everywhere
  `connection add` appears; gmail/gdrive/ga + s3/r2 **live commit** wired behind their clients
  (fail-closed without config); github/slack **networked reads** (`read_rows` + adapters).
- **Version** `0.0.8 → 0.0.9`.

## 4. Concerns

- **New vendor crypto dependency (deliberate, ticket-sanctioned).** The RustCrypto
  `p256`/`ecdsa`/`elliptic-curve`/`sec1`/`sha2`/`crypto-bigint` tree was added (t48 ES256 JWS for the
  OAuth AS; reused t80 for ECDH E2E wrap). It is the first non-trivial vendor crypto tree in an
  otherwise closed surface and was **not in the offline cache** (fetched from crates.io). Vetted,
  pure-Rust, wasm-clean; hand-rolling asymmetric crypto was explicitly avoided. `crypto-core` stays
  zero-dep. **Flag for reviewer acceptance.**
- **Documented seams — wired fail-closed, live half not claimed working** (network/native, not
  hermetically testable; consistent with the repo's honesty discipline and the parked-Cloudflare
  precedent): live OAuth browser consent (t49/t54), live MCP cloud tunnel dial (t63), live LDAP/AD/
  Entra/Workspace directory (t58), the qfs Cloud broker endpoint (t66), the payment provider (t67),
  Postgres/MySQL SQL backends (sql ticket — SQLite shipped), gmail/gdrive/ga/objstore **reads**
  (multi-shape — github/slack reads shipped), and the CF Workers wasm artifact (parked, ADR-0005).
  Each fails closed without config; none over-claims.
- **Carry-over / deferred-concern verdicts.** t57's actor-resolver wiring → **resolved** by t58's
  live `MembershipResolver` (fixture-backed; real directory a seam). t75's checker-inert risk →
  **resolved** (now active on the production path). t81's per-member differentiation awaits a richer
  `/sys/policies` subject column → **carries forward** (gate mechanism proven, admin surface is the
  gap). t80 forward-secrecy of a removed recipient → out of scope, as in any E2E system. The four
  superseded older tickets carry a recorded disposition note in their archived copy.
- **Open product decisions flagged in-ticket, not guessed:** DB/passphrase location (t42/t43),
  keyword case policy (t74 — chose case-insensitive/lowercase-canonical), payment provider (t67),
  admin-page shape (t53), broker token custody (t66), super-admin vs project-admin split (t53/t81).
- **Batch size.** 48 tickets / 90 commits in one release is large; mitigated by per-ticket green-bar
  verification and honest per-milestone status in `docs/roadmap.md`.
