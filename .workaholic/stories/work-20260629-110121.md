## 4. Outcome

This branch delivers three large epics plus a documentation-honesty program and a batch of optimizations/polish.

### Wire the binary so the docs run true (9-ticket epic)

- **Codec execution:** `decode`/`encode` pipeline stages now execute on the read path (JSON→YAML, CSV→JSON verified end-to-end); previously silently dropped at lowering.
- **Local content reads:** `/local/<file>` now emits a `content` (Bytes) column so codecs have bytes to transform.
- **Git reads:** `/git/<repo>[@<ref>]/{commits,tree,blob,refs,tags,reflog,blame}` read facet wired hermetically over a local `.git`; single-blob content reads and `@<ref>`/`@HEAD~n` time-travel added.
- **SQL reads:** `/sql/<conn>/<table>` read facet wired; WHERE, then ORDER BY + LIMIT, then column projection push into the database.
- **Cloud reads:** `/github`, `/slack`, `/mail` (Gmail label/root/single-message/attachment), `/drive` (folder walk + file content/export), `/ga` (Google Analytics report) reads all wired; fresh uncredentialed users now get an actionable connect-your-account capability error instead of `unknown_source`.
- **Error clarity:** `describe` native_verbs derive from a node's real Capabilities (no more advertising verbs the gate rejects); cloud-bind WARN noise downgraded to debug so unrelated runs are quiet.

### In-language connection declaration (9-ticket epic)

- **CREATE CONNECTION grammar** (`<name> DRIVER <driver> [AT '<locator>'] [SECRET '<ref>']`) added via contextual idents (no keyword-freeze change).
- **Secret references:** `env:<VAR>` and `vault:<driver>/<conn>` resolve lazily at bind time, secret-free on errors, backed by the existing encrypted store.
- **Registry from declarations:** `/sql` and `/git` mounts built from declared connections in `connections.qfs` (loaded via `QFS_CONNECTIONS` or `~/.config/qfs/`); the old `QFS_SQL_*`/`QFS_GIT_*` env-var convention demoted to a warned, `qfs connection import-env`-migratable fallback.

### Defined-paths epic — CONNECT replaces fixed mounts (4-ticket epic)

- **CONNECT/DISCONNECT statements** (contextual-ident sugar over a `/sys/paths` effect) persist path→driver bindings in the project DB (migration 8), readable via `/sys/paths` and `qfs connection paths`.
- **Recursive multi-segment paths** route via the existing longest-prefix router; CALL routes by canonical `id()` wherever a driver is mounted.
- **Fixed per-driver mounts removed:** on a fresh binary only `/local` and `/sys` resolve; every third-party driver is reachable only after CONNECT. `describe` still lists all driver TYPES so `drivers.md` is unchanged.

### Documentation rewritten to shipped reality

- Roadmap page erased then reborn as the one proposal-tagged page; README/index/guide/cookbook rewritten with examples verified against `target/debug/qfs`; every fenced qfs recipe parse-checked by `crates/test/tests/cookbook_skills.rs`.
- **gen-skills** added: each cookbook article generates one Agent Skill `SKILL.md` (never hand-edited, `--check`-gated); cookbook reorganized into per-service chapters; FTP-migration guide added then folded into per-service cookbooks.

### Live backends, optimizations, and polish

- Live **Postgres + MySQL** `/sql` backends with a podman dev stack; live **/s3 (MinIO)** reads; `/cf` and `/rest` mounted as reachable paths.
- Runtime-within-runtime panic fixed for all five networked cloud read facets (dedicated OS thread).
- Google OAuth **app credentials stored in qfs's own DB**; live Gmail/Drive reads verified against the owner's real account.
- Project-DB **migration heal** for a shipped in-place-edited v2 (checksum `1be5979f`) — opens and upgrades old DBs losslessly.

## 5. Historical Analysis

### Honesty discipline: docs are true or visibly incomplete

The branch's founding decision was to wire the binary rather than dumb the docs down. Every shipped example is verified against the real binary; generated docs (`language.md`/`drivers.md`/`server.md`) are never hand-edited and `gen-docs --check` gates anti-drift, while hand-authored prose gets a parse ratchet (`cookbook_skills.rs`) that flags any statement the binary would reject. This ratchet immediately caught seven real doc bugs (trailing semicolons, un-separated statements, `IS NOT NULL`, invalid `id:` selectors) when gen-skills landed.

### Infra usually already exists; registration is the gap

A recurring finding: codecs, git-object reading, SQL execution, and the Gmail/Drive/GitHub/Slack API clients were all implemented before this branch. The gap was read-facet registration in `shell.rs`, a dropped codec stage at pushdown lowering, and unwired PushdownProfile flags — not missing drivers. This shaped the phased, low-ripple approach across the wire-binary epic.

### Design-led epics resolve questions before code

Both the connection-declaration epic (keystone 004110) and the defined-paths epic (keystone 100010) began with a design ticket that settled grammar, mapping, secret scheme, and migration before any implementation. The defined-paths keystone was notably reshaped mid-flight by two owner corrections — qfs is experimental (no backward compat / no deprecation) and a connection is server state written by a side-effecting CONNECT, not a config file — which then propagated cleanly to every child ticket.

### Versioned grammar surface handled with contextual idents

Adding CREATE CONNECTION and CONNECT/DISCONNECT to the parser required additive care: the 39-keyword freeze is a hard constraint. Each new statement used contextual idents (keywords only in position) and desugared to existing effects (`/sys/paths`), so the closed Statement/EffectKind sets stayed unchanged — the pattern the earlier `AT` clause established.

### Residual correctness is load-bearing for pushdown

Across SQL, Gmail, and GA the same trap recurred: a native LIMIT must be dropped whenever a local residual remains (else it under-fetches before the re-filter), and a pushed projection must keep any column a residual predicate still needs. The engine re-applies only the unpushed work, so each facet must honor exactly its declared profile — flags flip on incrementally (WHERE → ORDER BY → LIMIT → projection).

### Fixed mounts → user-driven connections

The shift from env-var-named mounts to explicit CONNECT bindings made the project DB the single source of truth and required multi-connection routing keyed on canonical `id()`. The `/ga`→`/google-analytics` rename proved the `id()`-canonical decoupling (overriding `id()` kept existing connections from orphaning) before the epic relied on it.

## 6. Concerns

### Cloud reads panicked under runtime-within-runtime blocking

- **Severity:** moderate
- **Description:** Every cloud read facet's client drives the shared reqwest transport via its own `block_on`; called from inside the async read executor (itself a tokio worker) this panics with "Cannot start a runtime from within a runtime" (see [613c1f5] and [cf08355]). Only objstore was guarded, so gmail/gdrive/ga/github/slack live reads crashed the process; the hermetic mock-client path never exercised it. Fixed on this branch, but the class is easy to reintroduce.
- **How to Fix:** Run any blocking transport call on a dedicated OS thread (`std::thread::scope`) with no tokio context, reducing a panic to a structured secret-free error. Apply the same treatment to every future blocking-transport integration.

### project.db migration mismatch / store flakiness (203120)

- **Severity:** moderate
- **Description:** A pre-existing `~/.config/qfs/project.db` migration mismatch surfaced intermittently during live verification; the migration guide and live-verification tickets (see [30e5ca7], [cd41ddb]) each worked around it with a fresh `XDG_CONFIG_HOME`. The forward-heal for Project v2's `1be5979f` in-place edit ([9b46d6c]) fixed one known checksum, but the underlying issue was never confirmed-ticketed and remains open. The CONNECT epic's migration 8 raises the stakes since project.db is now the single source of truth for path bindings.

## 7. Successful Development Patterns

- **Anti-drift gates keep docs true.** `gen-docs` and `gen-skills` outputs are never hand-edited; `--check` gates fail CI on any drift, and the cookbook parse ratchet (`cookbook_skills.rs`) rejects any fenced qfs statement the binary can't parse. Testing the source articles (not the generated skills) keeps the check independent of generation, and it caught seven real doc bugs the moment gen-skills landed.
- **Narrow wiring over reimplementation.** When driver code already existed, the fix was registration (a read facet in `shell.rs`, one preserved stage in `lower.rs`, a flipped PushdownProfile flag) rather than new implementations — low-ripple and easy to verify against the real binary, letting the wire-binary epic ship as many small, independently-verifiable slices.
- **Hermetic tests + opt-in live verification.** Every cloud read is proven hermetically through a mock client in the default suite (fast, deterministic, no creds), while real-credential reads are verified out-of-band and recorded in ticket bodies. Splitting correctness-proof from integration-proof keeps CI reliable while still delivering credible real-world evidence.
- **Design keystone before implementation.** Large epics opened with a keystone ticket that settled grammar, mapping, secrets, and migration; children referenced it and parallelized. This absorbed the owner's mid-epic corrections with a single re-point rather than scattered rework.
- **Contextual idents preserve the keyword freeze.** CREATE CONNECTION and CONNECT/DISCONNECT parse `CONNECTION`/`DRIVER`/`SECRET` as contextual idents and desugar to existing `/sys/paths` effects, so the frozen keyword set and the closed Statement/EffectKind enums stay untouched — additive grammar growth without breaking any query that parses today.
- **Residual-correct, incrementally-deepened pushdown.** Each facet declares exactly what it pushes and the engine re-applies the rest; the load-bearing rule (drop a native LIMIT whenever a residual remains; keep projected columns a residual still needs) is wired once and reused. Flags flip on one at a time, bounding risk per slice and keeping regressions bisectable.
- **Off-runtime thread for blocking transports.** Blocking clients that own their own runtime are run on a dedicated OS thread via `std::thread::scope`, converting a would-be process-abort panic into a structured error. Established for objstore, then factored across the other five cloud facets.
- **Silent-wrong-answer as a bug class.** Several fixes targeted the same failure mode — plausible-but-wrong output rather than an error: codecs that no-op'd, describe advertising rejected verbs, git blob paths returning empty listings, EXTEND silently succeeding. Treating "plausible wrong output" as the worst failure and converting it to correct behaviour or a structured fail-closed error was a consistent quality lever.
