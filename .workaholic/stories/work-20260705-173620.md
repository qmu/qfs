# Branch story — `work-20260705-173620` (qfs v0.0.24)

15 commits over `main`. A multi-theme branch: the §13 declared-driver **tier-2** trio is completed,
the in-language **account** surface lands, an in-language **vault unlock** (`qfs auth`) lands, and a
**concern-triage** sweep converts the backlog into implemented fixes.

## 1. What this branch delivers

### §13 declared drivers — tier-2 complete
- **Transport honesty** (`0c825fe`): a dotted cursor path + a reqwest redirect-policy layer scoped to
  the declared driver's confined host — a 30x can no longer leave the boundary the `send_one` guard
  enforces.
- **View-body evaluation** (`fdfe534`, `4cef44b`): a declared view IS its stored query — reading a
  declared mount evaluates the matched view's body.
- **MAP `VALUES (<expr>)` apply-time evaluation** (`03bb96a`) + the Slack-twin **row-equivalence
  proof**.
- **Per-map `IRREVERSIBLE` now enforced at plan time** (`fbc97b8`, triage #1): the last named tier-2
  park. A new `Driver::write_irreversible` hook lets the planner set `EffectNode::irreversible` for a
  declared MAP marked IRREVERSIBLE, so PREVIEW surfaces it and COMMIT gates it behind
  `--commit-irreversible`, exactly like a `REMOVE`. **All three tier-1 parks are now closed.**

### Accounts — in-language, complete deletion
- **`CREATE ACCOUNT` + `/sys/accounts`** (`61f696c`): consent recorded (gated on a signed-in
  operator), a selectors-only registry (no token column, the Google trio collapsed to one row).
- **Filter-based removal** (`8933f7a`, triage #3): `REMOVE /sys/accounts WHERE account == '<email>'`
  removes a Google-email account in-language — the email rides as a string literal (data), so its `@`
  survives (a path would drop it). Provider resolved from the registry; ambiguous/absent fail closed.

### Vault — in-language unlock
- **`qfs auth`** (`baf3a9c`): warms the time-boxed session-unlock so the next command doesn't
  re-prompt (a short top-level operator verb).

### Reliability + tooling (concern triage)
- **Lazy commit-side apply bind** (`3d43e99`, triage #2): the apply registry no longer opens the
  credential store eagerly. A cloud mount whose quiet bind fails registers a `LazyApplyDriver` that
  binds at the first write — prompting on a terminal or erroring with an actionable locked-store /
  connect hint (never a silent "no driver"). Mirrors the read funnel; converges two carried concerns.
- **`/git @<ref>` nested-subtree reads** (`e8af6d8`, triage #4): `ls`/`cat`/`blame` over a nested path
  resolve as of any ref (was flat-tree E0 only).
- **`--` line comments in the connections parser** (`50f3db3`, triage #5): a `connections.qfs` can
  carry notes without silently dropping the next declaration.
- **`xtask check-migrations` pre-ship guard** (`81afad4`, triage #7): fails the build if a shipped
  migration body was edited in place without an audited `SUPERSEDED_BODIES` heal-forward entry.
  Wired into the anti-drift gate family.
