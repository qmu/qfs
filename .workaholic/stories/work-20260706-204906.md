# Branch story — `work-20260706-204906` (qfs v0.0.26)

1 commit over `main`. A single-purpose branch: it runs the **Conservative Vendor Dependence** audit
end to end and lands the durable outcome the policy asks for — a per-dependency **decision log** in
the one living design doc — plus the two hygiene actions the audit actually justified.

## 1. What this branch delivers

### The dependency-posture decision log (`84c28f5`, ticket `20260706194536`, commit `09739aa`)

The governing policy (`workaholic` `design/vendor-neutrality` + `implementation/vendor-neutrality`)
is *implement by default; take an external dependency only when a criterion is clearly met, and for
every taken dependency leave a Reason / Assessment / Monitoring / Exit record in the repo.* qfs
already discharged the "implement by default" half (in-house parser, combine engine, git reader,
HTTP server, test harness) and already nails the ACL half structurally (`driver-*` crates as
anti-corruption layers, pipe-SQL as the domain vocabulary, `tokio`/`futures`/`async-trait` confined
to `qfs-runtime` and enforced by `qfs-plan`'s purity dep-closure test). The measured gap was that the
~15 load-bearing *taken* deps had **no decision log**.

- **`docs/blueprint.md` §11** gains a `### Dependency posture — the decision log` section
  ([[blueprint-over-adr-pile]]: durable outcome lands in the one living blueprint, not a new ADR;
  git holds the audit reasoning). It logs the load-bearing set grouped by the criterion that admits
  each dep — *error-is-fatal / far-from-core* (crypto cluster, `time`, `keyring`, `rustix`),
  *interop / protocol conformance* (`reqwest`, `url`, the three DB drivers, `csv`), *cost/time
  efficiency* (`serde`, `toml`, `clap`, `tracing`, `tokio`, `winnow`), and *ergonomics — parked*
  (`thiserror`, `async-trait`, `base64`, `bytes`, `rpassword`) — each with Reason / Assessment /
  Monitoring / Exit. It records `async-trait` as a **monitored exit** (drop when native `dyn`
  async-trait dispatch is ergonomic) and explicitly parks the rest so a future reader does not
  re-litigate them. The measured framing is stated: direct third-party deps ≈ **29 shipped** (47
  internal crates), the transitive weight lives under `reqwest` + the three DB drivers + `tokio`,
  so the real reduction lever is *feature-trimming those*, not shaving convenience crates.

### The two hygiene actions the audit justified

- **Dead `anyhow` removed** — deleted from `packages/qfs/Cargo.toml` `[workspace.dependencies]`. It
  was declared but no member ever opted in (verified 2026-07-06): a free win, a dead line gone.
- **`serde_yaml` → `serde_yaml_ng`** — dtolnay archived `serde_yaml` at `0.9.34+deprecated` in 2024,
  a live development-status / sustainability red flag and the single clearest reduction signal the
  audit surfaced. Swapped to the maintained community fork (API-compatible drop-in), confined to
  `qfs-codec` (`codecs/yaml.rs`, `codecs/markdown.rs` frontmatter, and the `convert.rs` doc-comment
  bridge). The vendor type never crosses the `qfs-codec` public boundary — the codec still funnels
  through the shared `serde_json::Value` bridge — so the swap is **invisible to the query language**.
  The exit is recorded in the §11 log.

The audit named two genuine reduction candidates it deliberately did **not** execute here, both
captured in the ticket and blueprint §11 so they survive:

  transitive tree); verifiable by a `cargo tree` diff + green `cargo test --workspace`.
- **`tracing-subscriber` slim** — given the measured 87:9 flat-event:span ratio, a minimal formatter

wanted. Also noted (not in scope for the binary's log): the VitePress/npm docs toolchain is a
*separate* supply chain with its own advisories (3 npm vulns reported by `docker compose up docs`).
