## 1. Overview

Two shipped bugfixes, one refusal, and one filed seam. All three tickets came from the same triage batch; the third broke its premises under measurement and was stopped rather than forced.

**Span capture isolated** (`f93912b`). The runtime's observability tests flaked in parallel because a process-global capture (`OnceLock` + `set_global_default`) shared one buffer with every sibling in the test binary. The ticket's prescribed fix — `with_default` scoping — made things **worse** (7/40 failing, empty buffer), and probing disproved the comment that blamed the global max-level filter. The real mechanism is tracing's **global per-callsite `Interest` cache**: a sibling reaching `info_span!` before any subscriber is registered pins `Interest::never()` process-wide and silently switches the test off. Scoping cannot fix that while siblings share a process, so the capture moved to its own integration-test binary. Green 0/60 parallel, 0/20 serial. The secret-scan assertion was proven live both ways — a planted leak fails at exit 101, reverted passes at 0.

**REPL `/local` rooted at the session cwd** (`731fbc3`). Preview showed `local:/local/out.txt`; commit failed with `PermissionDenied` — and the error **is the proof of target**: `ec2-user` writes freely in the session's temp cwd, so the commit was aimed at filesystem root `/out.txt`. **As root it would have written to `/` silently.** The root became a constructor argument (`live_registry(local_root)`, `apply_plan_rooted`), with the `/`-rooted `apply_plan` kept as the delegate for job/mcp/sweeper. Pin watched red against the old root (exit 101), then green; the rebuilt binary lands `out.txt` in the session cwd.

**Describe-surface consistency: stopped, not archived.** All three sub-fixes failed their premises under measurement — see Concerns.

**The child-address seam filed** (`7b93018`, rulings recorded `4f3dacd`): describe declaring a row's child address per archetype, with the design session's settled rulings written in. Implemented on the follow-on branch `work-20260718-113000`, not here.

## 2. Motivation

Both shipped fixes are the family this repository keeps hunting: a well-formed answer that is wrong. A flaky observability suite is an instrument that lies intermittently — worthless for verifying anything else. A REPL that previews `local:/local/out.txt` and writes `/out.txt` is a preview that does not describe the commit — the exact contract (preview = what commit will do) the safety model stands on.

## 6. Concerns

**The describe-surface ticket (`20260716214500`) remains queued with three measured refutations recorded:**

1. *Navigable roots* — the ticket says children are "the listing `ls` already renders", but `ls /sys` and `ls /slack` **also error today**; its own gate requires per-root scan plumbing that does not exist, and `/slack`'s children are operator state needing a root-scoped read path with no client.
2. *Blob cd* — the ticket's "no stat during pure planning" constraint forces the read-side consult, which would make `cd /drive/x` a network+auth round-trip for **every** blob driver. Product decision.
3. *Slack `Rm`* — the stated mechanism is wrong (`decode_remove` already lowers `Files`+`id`); the real block is that `capability_verb_for` maps only Insert/Upsert/Update/Remove, so `Ls`/`Cp`/`Mv`/`Rm` are **requested by no statement anywhere** — a repo-wide dead-advert class across five drivers. Wiring it adds a new irreversible capability (product decision); a slack-only fix leaves the same falsehood on four other drivers.

Note: the enumerate facet settled in the address design (2026-07-18) gives refutation 1 its concept — that work is filed as `20260718120100` on the follow-on branch.

**Leftover to reconcile at merge time:** this branch still carries the child-address ticket at its `todo/` path; the follow-on branch archives the identical bytes. After both merge, the todo copy is the leftover to drop.

## 7. Successful Development Patterns

- **Watching the prescribed fix fail.** The ticket's own remedy made the flake worse, and that failure — not the original red — exposed the real mechanism.
- **Reading the error as evidence.** `PermissionDenied` was not noise to suppress but the proof of which path the commit actually targeted.
- **Stopping on measured premise collapse.** The third ticket was not massaged into something shippable; its three refutations are recorded where the next session will read them.
