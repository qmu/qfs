# Lexer / tokenizer

## Overview

This ticket delivers the **lexer** for the qfs surface syntax: the first stage of the
language core (RFD §3 "closed core + three open registries", §2.2 "Pipe-SQL"). It turns raw
statement bytes into a flat stream of typed tokens with byte **spans**, which the parser
(sibling ticket t04, built on the t02-selected combinator library — winnow by default,
RFD §9) consumes to produce the AST.

The lexer is the gatekeeper of the **frozen keyword set**: it must recognize exactly the
reserved UPPERCASE keywords and operators from RFD §3 and emit everything else as
identifiers, paths, or literals. Getting spans right here is what makes downstream
diagnostics ("unsupported verb at parse time" — RFD §5, §10) precise and AI-legible.
This stage is **pure** (`bytes -> Vec<Token>`, no I/O, no World), consistent with the
purity invariant (RFD §3).

## Scope

In scope:
- Tokenize the full closed-core surface: keywords, the `|>` pipe operator, comparison/logical
  operators, punctuation, identifiers, qualified names, paths, and all literal forms.
- Byte-offset **spans** (`start..end`) on every token for diagnostics.
- Keyword recognition against the frozen set (case-sensitive UPPERCASE); multi-word keywords
  (`GROUP BY`, `ORDER BY`, `INSERT INTO`, `MATERIALIZED VIEW`) handled as separate adjacent
  tokens — composition is the parser's job, not the lexer's.
- Literals: strings (`'…'` with escapes), numbers (int/float), **size literals** (`25 MB`),
  **typed literals** (`DATE '…'`), booleans, null.
- Path lexing: `/driver/seg/seg`, with the `@version` temporal coordinate (RFD §4) and glob
  characters surfaced as path tokens.
- Comments (line `--` and/or `#`, block) skipped but span-tracked.
- Lexer error type with span (unterminated string, stray char, bad escape).

Out of scope (deferred):
- All grammar/precedence/AST construction → **t04 parser**.
- Keyword/verb *capability* resolution (which verb a path supports) → driver/runtime tickets
  (resolved at parse/plan time against the driver contract, RFD §5; lexer only classifies).
- Function/procedure/codec **registry** resolution → registry tickets; the lexer emits bare
  identifiers, not registry lookups.
- Pushdown, effect-plan, PREVIEW/COMMIT semantics → E2 runtime tickets.

## Key components

New crate/module: `qfs-lang` crate, module `qfs_lang::lex` (the parser in t04 lives beside it
as `qfs_lang::parse`). SDK/vendor types never appear here — pure owned types only (RFD §9).

```rust
// span.rs — shared with parser
pub struct Span { pub start: u32, pub end: u32 }       // byte offsets into source
pub struct Spanned<T> { pub node: T, pub span: Span }

// token.rs
pub enum Token {
    // closed-core keywords (frozen; one variant per reserved word — RFD §3)
    Keyword(Keyword),                 // FROM WHERE SELECT … COMMIT, etc.
    // operators
    Pipe,                             // |>
    Eq, Ne, Lt, Gt, Le, Ge,          // = <> < > <= >=
    Tilde,                            // ~  (regex/match)
    // structural
    LParen, RParen, Comma, Dot, Star, Arrow, // ( ) , . *  =>
    // names & paths
    Ident(String),                    // bare identifier (driver, fn, column, proc seg)
    Path(Vec<PathSeg>),               // /driver/seg/seg(@version)
    // literals
    Str(String), Int(i64), Float(f64), Bool(bool), Null,
    Size { value: u64, unit: SizeUnit },     // 25 MB
    TypedLit { ty: LitType, raw: String },   // DATE '…', TIME '…', TIMESTAMP '…'
}

pub enum Keyword { /* exhaustive, generated from the RFD frozen list */ }
pub enum SizeUnit { B, KB, MB, GB, TB }
pub enum LitType { Date, Time, Timestamp }
pub struct PathSeg { pub name: String, pub version: Option<String>, pub glob: bool }

// lib api
pub fn lex(src: &str) -> Result<Vec<Spanned<Token>>, LexError>;
pub struct LexError { pub span: Span, pub kind: LexErrorKind }
pub enum LexErrorKind { UnterminatedString, BadEscape, UnexpectedChar(char), BadNumber }
```

Note: `LIKE ANY IN BETWEEN AND OR NOT` are **keyword** operators (lexed as `Keyword`),
matching RFD §3's operator list; only the symbolic operators get dedicated token variants.
`Arrow` (`=>`) supports named proc args (`CALL github.merge(method=>'squash')`, RFD §3).

## Implementation steps

1. Create the `qfs-lang` crate (workspace member); add `span.rs`, `token.rs`, `error.rs`,
   `lex.rs`; re-export from `lib.rs`.
2. Define `Span`/`Spanned` and `Token`/`Keyword`/`SizeUnit`/`LitType`/`PathSeg`.
3. Build the frozen-keyword table as a `match`/`phf`-style lookup; assert the table is the
   exact RFD §3 set in a unit test (drift guard).
4. Implement the core scanner (winnow if t02 confirms it, else hand-written byte cursor):
   skip whitespace + comments (span-tracked), dispatch on first byte.
5. Lex symbolic operators with longest-match (`|>`, `<=`, `>=`, `<>`, `=>` before single-char).
6. Lex identifiers `[A-Za-z_][A-Za-z0-9_]*`; classify UPPERCASE matches against the keyword
   table, else `Ident`.
7. Lex paths: leading `/` → consume segments, parse optional `@version` per segment, flag
   glob chars; keep raw segment strings (no driver validation here).
8. Lex string literals with escape handling; surface `UnterminatedString`/`BadEscape`.
9. Lex numbers; on a number followed by a size unit token, fold into `Token::Size`.
10. Lex typed literals: keyword (`DATE`/`TIME`/`TIMESTAMP`) immediately followed by a string →
    `TypedLit`; otherwise the keyword stands alone.
11. Attach spans to every emitted token; ensure spans are contiguous and cover comments.
12. Golden tests + property tests; wire `cargo fmt`/`clippy -D warnings` in CI.

## Considerations

- **Closed-core invariant**: the keyword table is the single chokepoint that keeps the grammar
  frozen (RFD §3). A test must fail if a keyword is added/removed without an RFD change. New
  backends must add **zero** keywords — verify the lexer has no per-driver branch.
- **No vendor leakage / owned types** (RFD §9): the lexer depends only on `std` (+ the parser
  combinator from t02). No service SDKs, no I/O — keeps it `wasm32`-clean.
- **Diagnostics first** (RFD §5/§10, AI-legibility): every token and every error carries a
  byte span; spans must round-trip to source slices exactly. This is load-bearing for the
  "rejected at parse time, structured error" promise that AI agents rely on.
- **Hard parts & resolution**:
  - *Path vs. division/glob ambiguity*: `/` always starts a path token at a statement/operator
    boundary; resolve by lexing `/`-led runs as paths and never emitting a division operator
    (qfs has no arithmetic `/` in the core grammar) — document this decision in the module.
  - *Size literal `25 MB` vs. `25` then ident `MB`*: handle in the lexer by peeking the next
    token only when the unit set matches `SizeUnit`; otherwise keep them separate so a column
    literally named `MB` is unaffected.
  - *Typed literal lookahead* (`DATE '…'`): bounded one-token lookahead; do **not** validate
    the date string contents (parser/runtime concern) — only capture `raw`.
  - *`@version` inside paths*: the `@` binds to the preceding segment; preserve raw ref text
    (could be a git ref, S3 versionId, drive rev — RFD §4) without interpretation.
- **Purity / testability**: `lex` is referentially transparent; this is the foundation for the
  whole dry-runnable pipeline (RFD §3 purity invariant). No secrets, no creds, no network —
  nothing to least-privilege here, but document that the lexer must never be fed credential
  material that could land in error messages (errors quote source spans).
- **Directory/coding standards**: one type per concern file, `#![deny(unsafe_code)]`,
  `clippy -D warnings`, public API documented with `///` and examples.

## Acceptance criteria

- `cargo build`, `cargo fmt --check`, and `cargo clippy -D warnings` are green for `qfs-lang`.
- Unit test asserts the keyword table equals the exact RFD §3 frozen set (drift guard).
- **Golden tests** over representative statements lex to the expected token+span stream,
  including:
  - `FROM /mail/inbox |> WHERE size > 25 MB AND subject ~ 'invoice' |> SELECT id, subject`
  - `FROM /git/repo@v1.2/src |> SELECT path`
  - `INSERT INTO /mail/drafts VALUES (…) RETURNING id`
  - `WHERE created BETWEEN DATE '2026-01-01' AND DATE '2026-06-20'`
  - `CALL github.merge(method=>'squash')`
  - `FROM /s3/bucket/*.json@latest`
- Every token's span slices back to the originating source substring (round-trip assertion).
- Error cases (unterminated string, stray `$`, bad escape) yield a `LexError` with the correct
  span and `LexErrorKind` — asserted by golden error tests.
- Property test: `lex` never panics on arbitrary UTF-8 input (returns `Ok`/`Err`, never aborts).
- No network/filesystem access and no credentials anywhere in the crate or its tests
  (no-live-creds by construction).
