# t68 ŌĆö First-class `fs` driver (real filesystem blob namespace)

## Overview

Delivers a first-class **`fs`** driver (M+, roadmap ┬¦1.2): the real filesystem as a blob namespace,
beyond today's `/local` sandbox. Where `/local` is a sandboxed convenience, `fs` addresses files on
the machine as ordinary paths under a `BlobNamespace` archetype, so an agent reads/writes real files
through the same grammar as every other backend. The driver is new but heavily templated: it mirrors
`crates/driver-local` end-to-end (`LocalFsDriver`/`Sandbox`/`scan_rows`/`LocalApplier`), differing in
that its root scoping is operator-configured rather than the fixed sandbox. It adds ZERO keywords (a
new path family is a registry edit). Because it widens blast radius beyond the sandbox, explicit root
scoping (no `..` escapes), `POLICY`, and preview are load-bearing ŌĆö this ticket treats that as the
core requirement, not an afterthought. Independent: it can land any time after the driver contract.

## Exact seams

- `crates/driver-local/` ŌĆö the template, copied structure-for-structure: `lib.rs` `LocalFsDriver`,
  `fs_core.rs` `Sandbox` (the root-scoping + `..`-escape guard ŌĆö the security heart, reused/hardened
  here), `read.rs` `scan_rows`, `row.rs`, `applier.rs` `LocalApplier`, `effect.rs`, and the apply
  bridge `local_apply_driver()` + `qfs_runtime::PlanApplierBridge`.
- `crates/driver/src/lib.rs` `pub trait Driver` + `Archetype::BlobNamespace` (the archetype `fs`
  declares), `Capabilities`, `Verb`, `NodeDesc`; introspective half stays pure (proof test
  `fixture_driver_introspection_is_pure`), only `applier()` impure.
- `crates/plan/src/node.rs` `EffectKind` (`Insert`/`Upsert`/`Update` reversible; `Remove`
  irreversible per `is_inherently_irreversible()`) ŌĆö deleting a real file is irreversible ŌåÆ extra
  acknowledgement; this is more consequential than in the sandbox, so do not soften it.
- `crates/core/src/registry.rs` `MountRegistry::{register,resolve,resolve_path}` ŌĆö register `/fs`.
  Describe registry: `crates/qfs/src/describe.rs` `describe_registry()`. Live apply registry:
  `crates/qfs/src/commit.rs` `live_registry()`.
- `crates/server/src/policy/enforce.rs` `evaluate` (pure, default-deny) ŌĆö `fs` reach must be
  `POLICY`-scopable (e.g. `ALLOW SELECT ON 'fs/projects/*'`); the wider blast radius makes this the
  primary guardrail.
- `crates/cmd/tests/dep_direction.rs` ŌĆö add `qfs-driver-fs` to the `qfs-driver-*` runtime-consumer
  allowlist; live wiring lands on `crates/qfs`.

## Implementation steps

1. New crate `qfs-driver-fs` templated on `crates/driver-local`: pure introspective half ŌĆö
   `describe` returns the `BlobNamespace` schema; copy and HARDEN `fs_core.rs` `Sandbox` so the root
   is an operator-configured allowlist of roots (not the fixed `/local` sandbox) with the same
   `..`-escape / symlink-escape rejection. Add a fixture-introspection purity test. Tree green (pure
   tests).
2. Read facet: implement `read.rs` `scan_rows` over a configured root ŌĆö listing/reading real files;
   `WHERE`/`SELECT` pushdown mirroring `driver-local`. The root is injected from the binary so the
   pure driver names no absolute path.
3. Write facet: implement the `LocalApplier`-style applier behind `qfs_runtime::PlanApplierBridge` for
   `INSERT`/`UPSERT`/`UPDATE` (reversible) and `REMOVE` (irreversible). Every write re-validates the
   resolved path is inside a configured root (defence-in-depth, not only at scan time).
4. Binary wiring: register `/fs` in `crates/qfs/src/describe.rs` `describe_registry()` (pure) and
   `crates/qfs/src/commit.rs` `live_registry()` (apply), injecting the operator-configured roots from
   `crates/qfs`. Add to `crates/cmd/tests/dep_direction.rs`.
5. Security tests: assert `..`, absolute-path injection, and symlink escapes outside the configured
   roots are rejected (no read, no write); assert `REMOVE /fs/...` lowers to an irreversible
   `EffectNode` requiring the extra acknowledgement; assert a `POLICY` scoped to one root denies
   another.
6. Docs + version: document `/fs` only after a live read + write + scoped-deny smoke passes;
   regenerate `docs/drivers.md` via `cargo run -p xtask -- gen-docs` (never hand-edit). Patch-bump
   `crates/qfs/Cargo.toml`.

## Considerations

- **Blast radius is the headline.** Unlike `/local`, `fs` can reach real files outside a sandbox, so
  the security floor is doing real work: explicit operator-configured root scoping, hard rejection of
  `..`/absolute/symlink escapes (validated at BOTH scan and apply time), `POLICY` scoping per root,
  and preview before commit. `REMOVE` is irreversible ŌåÆ extra acknowledgement; never reclassify it as
  reversible for convenience.
- **Purity discipline.** The introspective half names no path and does no I/O (roots injected from
  `crates/qfs`), keeping the driver wasm-buildable and the proof test green.
- **Zero new keywords.** A new path family is a registry + driver edit; `crates/lang/src/keywords.rs`
  stays frozen at 38.
- **Dep-direction.** New driver edges land on `crates/qfs`; add the crate to the allowlist in
  `crates/cmd/tests/dep_direction.rs`.
- **Open product decision to FLAG:** the default root policy when an operator configures none ŌĆö
  prefer *deny-all until a root is explicitly configured* (no implicit whole-disk access); name this
  default in the PR rather than shipping an open default.
- **Honesty-first + versioning:** keep `/fs` out of docs/skill until the smoke passes; own PR + patch
  bump in `crates/qfs/Cargo.toml` (currently 0.0.7) + `v0.0.x` tag on ship.
