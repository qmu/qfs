# Fix README.md — 3 of 4 Quickstart commands error verbatim

## Overview

`README.md` (the repo-root qfs README, the project's front door) has a **Quickstart whose commands a
copy-paste user cannot run** — 3 of 4 error against the `0.0.10` binary. Severity: **MISLEADING**
(install/model claims are sound; the quickstart isn't).

## Exact seams (verified, fresh user)

1. **Read example errors** (line 119): `qfs run "/mail/inbox |> where subject LIKE '%invoice%' |>
   select subject, from"` → `unknown_source: no read driver registered for source 'mail'` (exit 3).
   The comment calls it "PREVIEW … touches nothing", but it never previews.
2. **Insert uses an un-parseable placeholder** (lines 122-123): `insert into /mail/drafts values
   (...)` → `parse_error … UNKNOWN_KEYWORD` (the literal `(...)` is in a runnable `sh` block). Show a
   real tuple `values ('a@b.com','Hi','Body')`, which previews cleanly.
3. **Irreversible example uses invalid grammar** (line 126): `id:draft-1 |> call mail.send` →
   `parse_error … UNEXPECTED_CHAR`/`UNEXPECTED_TOKEN` — `id:draft-1` is not qfs grammar. Use
   `/mail/drafts |> where id == 'draft-1' |> call mail.send` (and note mail read/CALL is unwired).
4. **`--version` sample shows `-gnu`** (lines 140-146) — shipped releases are static **musl**
   (`unknown-linux-musl`), so a real installed binary never prints `-gnu`. Minor but a concrete-looking
   triple that matches no artifact. (Also the binary prints a 4th line `wasm32: …` not shown.)

## Implementation steps

1. **Rewrite the Quickstart to runnable commands** — lead with `qfs describe /local/<dir>` and a
   `/local` read that returns rows, then a write-plan preview with a real tuple; keep the `/mail`/CALL
   examples but mark them "needs a connection / read facet not yet wired" (foundation decision).
2. **Fix the placeholder + `id:` grammar** to valid forms.
3. **Correct the `--version` sample** to a musl triple + the `wasm32:` line (or genericize it).
4. **Refresh the surface lists** while here: the driver list omits the shipped `claude`/`directory`/
   `cf`/`http` drivers; verify the "shipped surface" section against the binary.

## Key files

- `README.md` (edit). Reference: `install.sh`, `crates/cmd/src/lib.rs`, `crates/qfs/src/version.rs`.

## Considerations

- The README is the first thing anyone runs — every fenced `sh` command must be copy-paste-runnable
  for a fresh user or explicitly marked as needing setup.
- The install/release/model sections were verified accurate — do not churn them.
