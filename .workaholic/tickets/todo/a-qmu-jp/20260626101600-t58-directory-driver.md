# t58 — `/directories/...` driver (LDAP / AD / Entra / Workspace)

## Overview

Delivers the external-directory half of **decision I** (roadmap §1.2) within milestone **M5**:
a new `/directories/...` driver exposes an external identity directory's **groups and users as
ordinary qfs paths**, so the `member_of('/directories/...')` predicate added in t57 can be driven
by an actual LDAP / Active Directory / Entra ID / Google Workspace directory — "drive one [the
policy] from the other [the directory]" (decision I). `/directories` is a reserved realm —
decision P / §1.3. This is a normal qfs driver: nothing about
the engine, grammar, or safety model is special-cased. The driver contract and the template to
copy already exist as libraries — `crates/driver/src/lib.rs` `trait Driver` and the
`crates/driver-local` reference implementation (`LocalFsDriver`/`LocalApplier`/`Sandbox`). The
Google-Workspace variant reuses the existing OAuth client. What is genuinely **new**: the
`qfs-driver-directory` crate, its path namespace (`/directories/<provider>/groups`, `.../users`),
and the membership-read backends. **Read-first** — listing groups/users/memberships, no
directory mutation in this slice.

## Exact seams

- `crates/driver/src/lib.rs` `trait Driver` (mount/id/describe/capabilities/procedures/pushdown/
  prelude/version_support/plan_write/applier) + `Archetype::RelationalTable` (groups/users/
  memberships are relational) + `Capabilities`/`Verb`/`NodeDesc`/`ProcSig`(`requires_scopes`).
  The introspective half stays **pure** (proof: `fixture_driver_introspection_is_pure`); only
  `applier()` is impure — and read-first means the applier is minimal/none for now.
- `crates/driver-local/` — the **template**: `lib.rs` `LocalFsDriver`, `read.rs` `scan_rows`,
  `row.rs`, and the `local_apply_driver()` + `qfs_runtime::PlanApplierBridge` apply-bridge
  pattern to mirror for any future write support.
- `crates/google-auth/src/lib.rs` — for the Workspace provider, reuse `OAuthClient`/
  `StoredTokenSource`/`GoogleApiClient` and the runtime-free `HttpExchange`; the one real
  `HttpTransport` is `crates/qfs/src/transport.rs`. LDAP/AD/Entra backends use their own
  thin clients (no heavy vendor SDK), mounted the same way.
- `crates/core/src/registry.rs` `MountRegistry::{register,resolve,resolve_path,drivers}` —
  registers `/directories/...`; `crates/qfs/src/describe.rs` `describe_registry()` (cred-free
  describe) and `crates/qfs/src/commit.rs` `live_registry()` (gated on
  `networked_credential(driver)`) — register the new driver in both, gated for live reads.
- **The t57 link:** t57's `MembershipResolver` is satisfied by reading
  `/directories/<provider>/groups/<g>/members` (or `/users/<u>/groups`) through this driver, so
  `member_of('/directories/...')` resolves against a live directory. This driver supplies the
  data; t57 owns the predicate and keeps `evaluate` pure (membership resolved into the context).
- Dep guard `crates/cmd/tests/dep_direction.rs` — add `qfs-driver-directory` to the
  runtime-consumer allowlist (alongside the other `qfs-driver-*`); the live edge lands on
  `crates/qfs`. `qfs-cmd` stays free of the driver crate.

## Implementation steps

1. **Scaffold the crate + pure introspection (green).** Create `qfs-driver-directory` copying
   `crates/driver-local`'s shape; implement the pure `Driver` half — `describe`/`capabilities`/
   schemas for `/directories/<provider>/groups`, `.../users`, and a memberships relation. Assert
   introspection purity (mirror `fixture_driver_introspection_is_pure`). No live backend yet.
2. **First read backend (mock).** Implement `scan_rows`-style reads for one provider (Google
   Workspace via `GoogleApiClient`, or LDAP) behind the `HttpExchange`/thin-client seam; golden
   tests against mock responses, no live directory.
3. **Register + gate.** Register the driver in `describe_registry()` (cred-free) and
   `live_registry()` (gated on `networked_credential`); add to the dep_direction allowlist.
   `describe /directories/...` works credential-free; live reads require a connection (t54-style
   consent for Workspace).
4. **Wire t57's resolver to the path.** Implement t57's `MembershipResolver` over a
   `/directories/...` read so `member_of('/directories/<provider>/groups/<g>')` resolves; integration
   test with a mock directory + a policy that grants on group membership.
5. **Honest docs + version.** Document `/directories/...` and the now-real `member_of` example in
   `docs/` (the generated `docs/drivers.md` via xtask, never hand-edited) only once the read path
   works. Bump patch in `crates/qfs/Cargo.toml`; run `cargo build/test/clippy/fmt` +
   `cargo run -p xtask -- gen-docs --check`.

## Key files

- New `crates/driver-directory/` (`lib.rs`, `read.rs`, `row.rs`) modelled on `crates/driver-local`.
- `crates/qfs/src/describe.rs` (`describe_registry`) and `crates/qfs/src/commit.rs`
  (`live_registry`) — register the new driver, gated.
- `crates/cmd/tests/dep_direction.rs` — add `qfs-driver-directory` to the allowlist.
- `crates/server/src/policy/enforce.rs` — t57's `MembershipResolver` impl reads via this driver.
- `docs/drivers.md` is generated — change the source, regenerate via xtask; never hand-edit.

## Considerations

- **Safety floor, read-first.** Describe is pure and credential-free; reads touch nothing
  mutable. Deferring directory *writes* keeps blast radius minimal — provisioning/deprovisioning
  identities is a much larger risk surface and is out of scope here (flag as a future ticket with
  its own preview/commit/irreversible analysis).
- **Authz is upstream of this data — beware the loop.** A directory read that *feeds* a policy
  decision must itself be safe to perform without that policy gating it into a cycle; resolve
  membership through a path that does not re-enter the same `member_of` evaluation. Keep t57's
  `evaluate` pure by resolving membership *before* the decision, not inside it.
- **Secrets discipline.** Directory bind credentials / OAuth tokens are `qfs_secrets::Secret`,
  resolved lazily via `networked_credential`, never logged. Workspace reuses the t54 consent
  flow; LDAP/AD bind secrets go through the t43 envelope-encrypted store.
- **Owned DTOs + purity.** No vendor directory type leaks past the driver boundary; the
  introspective half stays pure and wasm-buildable; the live read lands on `crates/qfs`.
- **Dep-direction.** New driver crate added to the dep_direction allowlist; live edge on the
  binary leaf only; `qfs-cmd` stays clean.
- **Open decision (flag).** Which provider ships first (Workspace vs. LDAP/AD vs. Entra) and the
  canonical group/user path shape (`/directories/<provider>/groups/<g>/members` vs. a flat
  memberships relation) are worth pinning in review rather than guessing — they shape the t57
  `member_of` argument syntax.
- **Versioning.** One PR + patch bump in `crates/qfs/Cargo.toml` + `v0.0.x` tag on ship.
